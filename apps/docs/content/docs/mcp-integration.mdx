title: MCP integration
description: Connect Sendero tools to Claude Desktop, Cursor, Zed, or your own MCP-compatible agent host.
---

Sendero exposes a Streamable HTTP MCP server at:

```text
https://edge.sendero.travel/mcp
```

The Next.js app also serves the same protocol at:

```text
https://app.sendero.travel/api/mcp
```

Both endpoints derive tools from `@sendero/tools`, so web chat, WhatsApp, Slack, and MCP share one registry.

## Protocol

The MCP surface speaks JSON-RPC 2.0. Start with `initialize`, then `tools/list`, then `tools/call`.

```bash
curl -X POST https://edge.sendero.travel/mcp \
  -H "content-type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "initialize",
    "params": {}
  }'
```

```bash
curl -X POST https://edge.sendero.travel/mcp \
  -H "content-type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 2,
    "method": "tools/list"
  }'
```

Call a tool:

```bash
curl -X POST https://edge.sendero.travel/mcp \
  -H "content-type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 3,
    "method": "tools/call",
    "params": {
      "name": "search_flights",
      "arguments": {
        "origin": "MEX",
        "destination": "SCL",
        "departureDate": "2026-05-15",
        "passengers": 2,
        "cabinClass": "economy"
      }
    }
  }'
```

MCP tool results are returned as text content containing formatted JSON. Parse that JSON before threading values such as offer ids, trip ids, or booking ids into later calls.

## Claude Desktop

If your MCP host can connect to remote Streamable HTTP servers directly, point it at `https://edge.sendero.travel/mcp`.

If your host expects a local command, wrap the remote endpoint with your preferred MCP proxy. Keep these environment variables available to the proxy:

```json
{
  "SENDERO_EDGE_URL": "https://edge.sendero.travel",
  "SENDERO_API_KEY": "sk_test_..."
}
```

After restart, ask the host to list Sendero tools. Then ask for a scoped action such as:

> Search flights from Mexico City to Santiago next Friday for two passengers.

For production booking, have the host follow [agent-to-agent booking](/docs/agent-to-agent-booking) and prefer the named `sendero.book_flight` workflow where available.

## Cursor, Zed, and custom agents

Custom agents should:

1. Read `https://edge.sendero.travel/.well-known/llms.txt`.
2. Initialize MCP at `https://edge.sendero.travel/mcp`.
3. Call `tools/list`.
4. Select tools by schema, not by prompt text.
5. Use workflow APIs for multi-step booking, approvals, ticketing waits, settlement, and invoices.

## Direct HTTP fallback

Some agent platforms do not speak MCP. Use the x402-gated HTTP surface instead:

```text
GET  https://edge.sendero.travel/tools
POST https://edge.sendero.travel/tools/search_flights
```

Those endpoints use the `PAYMENT-REQUIRED` and `Payment-Signature` headers described in [x402 nanopayments](/docs/x402-nanopayments).

## Payment notes

MCP is the schema and tool-call interface. Direct paid HTTP endpoints are the x402 interface. Public deployments can enforce API keys or x402 at the edge, so clients should always send an `Authorization: Bearer ...` header even when local development routes do not inspect it.

For irreversible actions, the calling agent must get user or operator approval before booking, paying, cancelling, or changing travel.
