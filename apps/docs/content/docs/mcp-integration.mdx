Sendero exposes a Streamable HTTP MCP server. Two endpoints, one tool registry:

```text
https://app.sendero.travel/api/mcp        # canonical (Next.js app)
https://edge.sendero.travel/mcp           # edge mirror (Cloudflare Worker)
```

Both endpoints derive tools from `@sendero/tools`, so web chat, WhatsApp, Slack, MCP, and direct HTTP share one registry. Pick whichever URL is closer to your client.

## Pick your client

Each tab below has a one-click or one-line install. The auth header is the same across every client: a Clerk-issued API key minted at [`/dashboard/settings/api-keys`](https://app.sendero.travel/dashboard/settings/api-keys) (sandbox keys are auto-issued at workspace creation).

<Tab value="Claude Desktop">

Claude Desktop ships a one-click `.mcpb` bundle that wraps the remote MCP endpoint with a local stdio→HTTP proxy. Install once, paste a key, done.

```text
https://app.sendero.travel/downloads/sendero.mcpb
```

That URL 302s to the latest GitHub Release artifact (rebuilt every plugin tag). Open the downloaded file in Claude Desktop → Settings → Extensions → Sendero → Configure, paste your `ak_…` key, and the server appears in the picker.

If you'd rather not use the bundle, the plain JSON config is:

```json
{
  "mcpServers": {
    "sendero": {
      "type": "http",
      "url": "https://app.sendero.travel/api/mcp",
      "headers": { "Authorization": "Bearer ak_..." }
    }
  }
}
```

After restart, ask Claude something like *"Use Sendero to search flights from Mexico City to Santiago next Friday for two passengers."*

</Tab>

<Tab value="Claude Code">

For the Claude Code CLI, register Sendero with one command:

```bash
claude mcp add sendero \
  --transport http \
  --url https://app.sendero.travel/api/mcp \
  --header "Authorization: Bearer ak_..."
```

Or — install the versioned **Sendero plugin** that bundles the MCP config plus a `/sendero:travel-booking` skill that teaches Claude when to use Sendero:

```bash
git clone https://github.com/tcxcx/sendero.git
export SENDERO_API_KEY=ak_your_key
claude --plugin-dir ./sendero/apps/claude-code-plugin
```

See [Claude Code plugin](/docs/claude-code-plugin) for the full plugin install guide.

<Tab value="Custom agent">

Bringing your own agent? The protocol is JSON-RPC 2.0 over HTTP. Read [`/.well-known/llms.txt`](https://docs.sendero.travel/.well-known/llms.txt) for the discovery contract, then call:

```bash
# Initialize
curl -X POST https://app.sendero.travel/api/mcp \
  -H "content-type: application/json" \
  -H "Authorization: Bearer ak_..." \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{}}'

# List tools
curl -X POST https://app.sendero.travel/api/mcp \
  -H "content-type: application/json" \
  -H "Authorization: Bearer ak_..." \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/list"}'

# Call a tool
curl -X POST https://app.sendero.travel/api/mcp \
  -H "content-type: application/json" \
  -H "Authorization: Bearer ak_..." \
  -d '{
    "jsonrpc":"2.0","id":3,"method":"tools/call",
    "params":{
      "name":"search_flights",
      "arguments":{"origin":"MEX","destination":"SCL","departureDate":"2026-05-15","passengers":2}
    }
  }'
```

Custom agents should:

1. Read [`/.well-known/llms.txt`](https://docs.sendero.travel/.well-known/llms.txt).
2. Initialize MCP at `https://app.sendero.travel/api/mcp`.
3. Call `tools/list` once.
4. Select tools by schema, not by prompt text.
5. Use workflow APIs for multi-step booking, approvals, ticketing waits, settlement, and invoices.

</Tab>

## Protocol notes

MCP tool results return as text content containing formatted JSON. Parse that JSON before threading values such as offer ids, trip ids, or booking ids into later calls.

For irreversible actions (booking, paying, cancelling, changing travel), the calling agent must get user or operator approval before the call. The recommended pattern is `hold` → present to user → `confirm_booking` only on explicit OK.

## Direct HTTP fallback (no MCP)

Some agent platforms do not speak MCP. Use the x402-gated HTTP surface instead:

```text
GET  https://edge.sendero.travel/tools
POST https://edge.sendero.travel/tools/search_flights
```

Those endpoints use the `PAYMENT-REQUIRED` and `Payment-Signature` headers described in [x402 nanopayments](/docs/x402-nanopayments).

## Auth at a glance

MCP is the schema and tool-call interface; direct paid HTTP endpoints are the x402 interface. Public deployments can enforce API keys or x402 at the edge, so clients should always send an `Authorization: Bearer ...` header even when local development routes do not inspect it.
